# 配置文件管理规范

## 配置文件格式与位置

### 1. 配置表文件

**位置**: `resources/excel_tables/` 目录
**格式**: Excel (.xlsx)
**文件命名**: 小写蛇形命名

| 配置表 | 文件名 | 用途 |
|--------|--------|------|
| 技能配置 | skill.xlsx | 技能属性、效果、冷却等 |
| 物品配置 | item.xlsx | 物品属性、效果、价格等 |
| 宠物配置 | pet.xlsx | 宠物属性、技能、进化等 |
| 坐骑配置 | mount.xlsx | 坐骑属性、技能、进化等 |
| 任务配置 | quest.xlsx | 任务条件、奖励、流程等 |
| 商店配置 | shop.xlsx | 商店物品、价格、库存等 |
| 成就配置 | - | 成就条件、奖励、进度等 |
| 地图配置 | map.xlsx | 地图属性、区域、事件等 |
| 怪物配置 | monster.xlsx | 怪物属性、AI、掉落等 |
| 出生点配置 | spawn_point.xlsx | 怪物刷新点、数量等 |
| 玩家等级 | player_level.xlsx | 等级经验、属性成长等 |
| BUFF配置 | buff.xlsx | BUFF效果、持续时间等 |
| 公会配置 | guild.xlsx | 公会等级、权限、技能等 |
| AI配置 | ai.xlsx | AI行为、决策树等 |

### 2. 地图文件

**位置**: `resources/maps/` 目录
**格式**: JSON
**文件命名**: 地图名称.json

### 3. 协议文件

**位置**: `resources/protocol/` 目录
**格式**: Protocol Buffers (.proto)

## 代码加载规则

### 1. 配置表加载

- 使用 `zMmoShared/config/tables/` 中的表格管理类加载配置
- 加载顺序: 启动时一次性加载所有配置表
- 缓存机制: 配置加载后缓存到内存，避免重复读取

### 2. 地图文件加载

- 使用 `zMmoShared/config/tables/map_table.go` 加载地图配置
- 地图数据通过 ID 索引，支持动态加载

### 3. 配置更新

- 配置表更新后需要重启服务器才能生效
- 地图文件支持热更新（需要实现重载机制）

## 注意事项

1. **禁止在其他目录创建配置文件**
   - 所有配置文件必须放在指定目录
   - 禁止在 `MapServer/config/` 等目录创建JSON配置文件

2. **配置格式规范**
   - Excel表格必须有正确的表头
   - JSON文件必须符合语法规范
   - 所有配置项必须有默认值

3. **配置版本管理**
   - 配置文件变更需要记录版本号
   - 重大变更需要进行兼容性处理

4. **错误处理**
   - 配置加载失败时必须记录详细日志
   - 关键配置缺失时服务器应拒绝启动

5. **性能优化**
   - 配置加载应在启动时完成，避免运行时IO操作
   - 频繁访问的配置应使用缓存

## 迁移指南

### 从旧配置迁移到新配置

1. **停止服务器**
2. **备份旧配置**
3. **将配置转换为Excel格式**
4. **放入 `resources/excel_tables/` 目录**
5. **修改代码加载逻辑**
6. **启动服务器验证**

### 配置验证

启动时应验证以下内容:
- 所有必需配置表是否存在
- 配置格式是否正确
- 配置项是否完整
- 引用关系是否正确
