# zMmoServer 项目实施方案

## 一、项目现状分析

### 1.1 现有项目结构

```
D:\GitHub\
├── zEngine/           # 游戏引擎库（网络、日志、Actor、服务发现等）
```

## 二、项目整体规划

### 2.1 项目仓库结构

```
D:\GitHub\
├── zEngine/              # 游戏引擎（保持不变，继续完善）
├── zUtil/                # 通用工具库（新建）
├── zMmoServer/           # MMO项目根目录
│   ├── docs/             # 文档（已有）
│   ├── zMmoShared/       # MMO共享代码库（新建）
│   ├── GlobalServer/     # 全局服务器
│   ├── GatewayServer/    # 网关服务器
│   ├── GameServer/       # 游戏逻辑服务器
│   ├── MapServer/        # 地图服务器
│   ├── AdminServer/      # 后台管理服务器
│   ├── ChatServer/       # 聊天服务器（可选，后期）
│   ├── MatchServer/      # 匹配服务器（可选，后期）
│   ├── deploy/           # 部署相关（Docker、K8s）
│   ├── tools/            # 工具（Excel导出、协议生成等）
│   └── resources/        # 资源配置
│       ├── excel_tables/ # Excel配置表
│       ├── maps/         # 地图数据
│       ├── navmesh/      # 寻路数据
│       └── protocol/     # Protobuf协议定义
```

### 2.2 模块依赖关系

```
GlobalServer, GatewayServer, GameServer, MapServer, AdminServer
    ↓
zMmoShared (MMO共享代码)
    ↓
zEngine (游戏引擎) + zUtil (通用工具)
```

## 三、各模块职责划分

### 3.1 zEngine - 游戏引擎库

**职责：** 提供底层基础设施，与具体游戏逻辑无关

**保留的模块：**
- ✅ zNet - 网络层
- ✅ zLog - 日志系统
- ✅ zActor - Actor模型
- ✅ zEtcd - etcd服务发现
- ✅ zEvent - 事件总线
- ✅ zInject - 依赖注入
- ✅ zNavMap - 寻路
- ✅ zObject - 对象池、对象管理
- ✅ zScript - 脚本系统
- ✅ zService - 服务管理
- ✅ zSignal - 信号处理
- ✅ zSystem - 系统管理

**需要增强的模块：**
- zEtcd - 增加配置中心功能
- zLog - 增加日志写入数据库功能

## 六、关键技术点

### 6.1 服务间通信

- 协议：Protobuf
- 传输：TCP
- 服务发现：etcd
- 通信框架：可基于 zNet 扩展

### 6.2 数据一致性

- 玩家数据：GameServer 负责，定期持久化
- 跨服数据：GlobalServer 负责
- 最终一致性：业务逻辑保证

### 6.3 高可用

- 无状态服务：GatewayServer、GameServer、MapServer 可水平扩展
- 有状态服务：GlobalServer 需做主从
- etcd 集群：保证服务发现高可用

### 6.4 性能优化

- AOI：网格算法，减少广播
- 缓存：多层缓存，减少数据库访问
- 协程池：控制并发
- 对象池：减少GC
