## 一、方案概述

### 1.1 设计目标

- **服务自动发现**：各服务器实例启动后自动注册到服务注册中心，其他服务可以动态发现可用的服务实例
- **健康状态监控**：实时监控服务健康状态，自动剔除不健康的服务实例
- **负载均衡**：支持多种负载均衡策略，合理分配请求
- **故障自动转移**：当服务实例故障时，自动将请求转移到健康实例
- **高可用性**：服务注册中心自身具备高可用性，避免单点故障
- **客户端不接入**：客户端不直接参与服务发现，通过Gateway和GameServer进行通信

### 1.2 技术选型

| 组件         | 技术           | 版本   | 选型理由                                         |
| ------------ | -------------- | ------ | ------------------------------------------------ |
| 服务注册中心 | etcd           | 3.5.0+ | 分布式一致性存储，高可用，支持服务发现和配置管理 |
| 服务间通信   | TCP + Protobuf | -      | 高性能，低延迟，适合游戏服务器间通信             |
| 健康检查     | HTTP + gRPC    | -      | 支持多种健康检查方式，灵活可靠                   |

## 二、服务注册

### 2.1 注册机制

- **注册时机**：服务启动时自动注册，服务停止时自动注销
- **注册内容**：
  - 服务类型（GameServer/MapServer/GatewayServer/GlobalServer）
  - 服务ID
  - 服务组ID
  - 服务地址（IP:端口）
  - 服务状态（healthy/unhealthy/maintenance）
  - 服务负载信息（CPU/内存使用率、在线玩家数）
  - 服务版本
  - 注册时间

### 2.2 注册路径

```
/services/{service_type}/{group_id}/{service_id}
```

**示例**：

- GameServer 注册路径：`/services/gameserver/1/000101`
- MapServer 注册路径：`/services/mapserver/1/1`
- GatewayServer 注册路径：`/services/gateway/1/000101`
- GlobalServer 注册路径：`/services/globalserver/1/1`

### 2.3 注册数据格式

```json
{
  "id": "000101",
  "group_id": "1",
  "name": "GameServer-000101",
  "address": "192.168.1.100",
  "port": 20001,
  "status": "healthy",
  "load": 0.3,
  "players": 1234,
  "version": "1.0.0",
  "registered_at": 1678765432,
  "last_heartbeat": 1678765432
}
```

### 2.4 注册实现

- 使用 etcd 的 `Put` API 进行注册
- 设置键的TTL（生存时间）为 30 秒
- 服务启动时注册，之后每 10 秒发送一次心跳，更新 TTL
- 服务停止时使用 `Delete` API 注销

## 三、服务发现

### 3.1 发现机制

- **主动发现**：服务启动时主动查询服务注册中心，获取可用的服务实例
- **被动发现**：通过 etcd 的 `Watch` API 监听服务变化，实时更新服务列表
- **缓存机制**：本地缓存服务列表，减少对 etcd 的频繁查询

### 3.2 发现路径

- **按服务类型发现**：`/services/{service_type}`
- **按服务组发现**：`/services/{service_type}/{group_id}`
- **按服务ID发现**：`/services/{service_type}/{group_id}/{service_id}`

### 3.3 发现实现

- 使用 etcd 的 `Get` API 获取服务列表
- 使用 etcd 的 `Watch` API 监听服务变化
- 本地维护服务列表缓存，定期更新
- 支持根据服务状态、负载等条件过滤服务实例

## 四、健康检查

### 4.1 检查机制

- **心跳检查**：服务定期向 etcd 发送心跳，更新 TTL
- **HTTP 健康检查**：提供 HTTP 健康检查接口，返回服务状态
- **gRPC 健康检查**：提供 gRPC 健康检查接口，返回服务状态

### 4.2 检查内容

- **服务状态**：服务是否正常运行
- **资源使用**：CPU、内存使用率
- **连接状态**：连接数、连接状态
- **业务指标**：在线玩家数、消息处理延迟

### 4.3 检查频率

- **心跳**：每 10 秒一次
- **HTTP 检查**：每 30 秒一次
- **gRPC 检查**：每 30 秒一次

### 4.4 检查实现

- 服务启动时注册健康检查接口
- 定期执行健康检查，更新服务状态
- 当服务状态变为不健康时，自动更新注册信息
- etcd 会自动剔除超过 TTL 未心跳的服务

## 五、负载均衡

### 5.1 负载均衡策略

- **轮询**：按顺序依次选择服务实例
- **随机**：随机选择服务实例
- **最少连接**：选择当前连接数最少的服务实例
- **最低负载**：选择当前负载最低的服务实例
- **哈希**：根据请求参数哈希选择服务实例（如玩家ID）

### 5.2 负载均衡实现

- 服务发现模块返回可用服务列表
- 负载均衡模块根据选择的策略选择服务实例
- 支持动态调整负载均衡策略
- 支持服务权重配置

## 六、故障处理

### 6.1 故障检测

- **心跳超时**：服务超过 TTL 未发送心跳，视为故障
- **健康检查失败**：健康检查连续失败，视为故障
- **连接失败**：尝试连接服务失败，视为故障

### 6.2 故障处理流程

1. **故障检测**：检测到服务故障
2. **服务剔除**：将故障服务从可用服务列表中剔除
3. **故障转移**：将请求转移到健康服务实例
4. **故障恢复**：当故障服务恢复后，重新加入可用服务列表

### 6.3 故障处理实现

- 定期检查服务状态
- 当服务状态变为不健康时，从本地缓存中移除
- 当服务恢复健康时，重新添加到本地缓存
- 支持故障服务的自动恢复检测

## 八、代码实现

### 8.1 服务注册与发现核心代码

```go
// ServiceRegistry 服务注册器
type ServiceRegistry struct {
    etcdClient  *clientv3.Client
    serviceInfo *ServiceInfo
    leaseID     clientv3.LeaseID
    keepAliveCh <-chan *clientv3.LeaseKeepAliveResponse
}

// ServiceInfo 服务信息
type ServiceInfo struct {
    ID            string    `json:"id"`
    GroupID       string    `json:"group_id"`
    Name          string    `json:"name"`
    Address       string    `json:"address"`
    Port          int       `json:"port"`
    Status        string    `json:"status"`
    Load          float64   `json:"load"`
    Players       int       `json:"players"`
    Version       string    `json:"version"`
    RegisteredAt  int64     `json:"registered_at"`
    LastHeartbeat int64     `json:"last_heartbeat"`
}

// Register 注册服务
func (r *ServiceRegistry) Register() error {
    // 创建租约
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()
  
    resp, err := r.etcdClient.Grant(ctx, int64(r.serviceInfo.TTL))
    if err != nil {
        return err
    }
  
    r.leaseID = resp.ID
  
    // 注册服务
    key := fmt.Sprintf("/services/%s/%s/%s", r.serviceInfo.Type, r.serviceInfo.GroupID, r.serviceInfo.ID)
    value, err := json.Marshal(r.serviceInfo)
    if err != nil {
        return err
    }
  
    ctx, cancel = context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()
  
    _, err = r.etcdClient.Put(ctx, key, string(value), clientv3.WithLease(r.leaseID))
    if err != nil {
        return err
    }
  
    // 启动心跳
    ctx, cancel = context.WithCancel(context.Background())
    r.keepAliveCh, err = r.etcdClient.KeepAlive(ctx, r.leaseID)
    if err != nil {
        return err
    }
  
    // 处理心跳响应
    go func() {
        for range r.keepAliveCh {
            // 心跳成功
        }
    }()
  
    return nil
}

// Unregister 注销服务
func (r *ServiceRegistry) Unregister() error {
    key := fmt.Sprintf("/services/%s/%s/%s", r.serviceInfo.Type, r.serviceInfo.GroupID, r.serviceInfo.ID)
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()
  
    _, err := r.etcdClient.Delete(ctx, key)
    if err != nil {
        return err
    }
  
    return nil
}

// ServiceDiscoverer 服务发现器
type ServiceDiscoverer struct {
    etcdClient *clientv3.Client
    cache      map[string][]*ServiceInfo
    mu         sync.RWMutex
}

// Discover 发现服务
func (d *ServiceDiscoverer) Discover(serviceType, groupID string) ([]*ServiceInfo, error) {
    key := fmt.Sprintf("/services/%s/%s", serviceType, groupID)
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()
  
    resp, err := d.etcdClient.Get(ctx, key, clientv3.WithPrefix())
    if err != nil {
        return nil, err
    }
  
    var services []*ServiceInfo
    for _, kv := range resp.Kvs {
        var service ServiceInfo
        if err := json.Unmarshal(kv.Value, &service); err != nil {
            continue
        }
        if service.Status == "healthy" {
            services = append(services, &service)
        }
    }
  
    // 更新缓存
    d.mu.Lock()
    d.cache[key] = services
    d.mu.Unlock()
  
    return services, nil
}

// Watch 监听服务变化
func (d *ServiceDiscoverer) Watch(serviceType, groupID string) {
    key := fmt.Sprintf("/services/%s/%s", serviceType, groupID)
    watchChan := d.etcdClient.Watch(context.Background(), key, clientv3.WithPrefix())
  
    for watchResp := range watchChan {
        for _, event := range watchResp.Events {
            // 处理服务变化事件
            d.Discover(serviceType, groupID)
        }
    }
}
```

### 8.2 健康检查实现

```go
// HealthChecker 健康检查器
type HealthChecker struct {
    serviceInfo *ServiceInfo
    checkInterval time.Duration
}

// Start 启动健康检查
func (h *HealthChecker) Start() {
    ticker := time.NewTicker(h.checkInterval)
    defer ticker.Stop()
  
    for range ticker.C {
        h.check()
    }
}

// check 执行健康检查
func (h *HealthChecker) check() {
    // 检查服务状态
    status := "healthy"
  
    // 检查CPU使用率
    cpuUsage := getCPUUsage()
    if cpuUsage > 80 {
        status = "unhealthy"
    }
  
    // 检查内存使用率
    memUsage := getMemUsage()
    if memUsage > 80 {
        status = "unhealthy"
    }
  
    // 检查连接数
    connCount := getConnCount()
    if connCount > h.serviceInfo.MaxConnections * 0.9 {
        status = "unhealthy"
    }
  
    // 更新服务状态
    h.serviceInfo.Status = status
    h.serviceInfo.Load = cpuUsage / 100
    h.serviceInfo.LastHeartbeat = time.Now().Unix()
}
```

### 8.3 负载均衡实现

```go
// LoadBalancer 负载均衡器
type LoadBalancer struct {
    services []*ServiceInfo
    strategy string
    index    int
    mu       sync.Mutex
}

// NewLoadBalancer 创建负载均衡器
func NewLoadBalancer(strategy string) *LoadBalancer {
    return &LoadBalancer{
        strategy: strategy,
        index:    0,
    }
}

// UpdateServices 更新服务列表
func (lb *LoadBalancer) UpdateServices(services []*ServiceInfo) {
    lb.mu.Lock()
    defer lb.mu.Unlock()
    lb.services = services
}

// Next 选择下一个服务
func (lb *LoadBalancer) Next() *ServiceInfo {
    lb.mu.Lock()
    defer lb.mu.Unlock()
  
    if len(lb.services) == 0 {
        return nil
    }
  
    switch lb.strategy {
    case "round_robin":
        return lb.roundRobin()
    case "random":
        return lb.random()
    case "least_connections":
        return lb.leastConnections()
    case "least_load":
        return lb.leastLoad()
    default:
        return lb.roundRobin()
    }
}

// roundRobin 轮询策略
func (lb *LoadBalancer) roundRobin() *ServiceInfo {
    service := lb.services[lb.index]
    lb.index = (lb.index + 1) % len(lb.services)
    return service
}

// random 随机策略
func (lb *LoadBalancer) random() *ServiceInfo {
    index := rand.Intn(len(lb.services))
    return lb.services[index]
}

// leastConnections 最少连接策略
func (lb *LoadBalancer) leastConnections() *ServiceInfo {
    var bestService *ServiceInfo
    minConnections := math.MaxInt32
  
    for _, service := range lb.services {
        if service.Players < minConnections {
            minConnections = service.Players
            bestService = service
        }
    }
  
    return bestService
}

// leastLoad 最低负载策略
func (lb *LoadBalancer) leastLoad() *ServiceInfo {
    var bestService *ServiceInfo
    minLoad := 1.0
  
    for _, service := range lb.services {
        if service.Load < minLoad {
            minLoad = service.Load
            bestService = service
        }
    }
  
    return bestService
}
```

## 十、总结

本服务发现方案基于 etcd 实现，提供了完整的服务注册、发现、健康检查、负载均衡和故障处理机制。该方案具有以下优点：

1. **高可用性**：etcd 集群保证服务注册中心的高可用性
2. **实时性**：通过 Watch 机制实时感知服务变化
3. **可靠性**：多重健康检查确保服务状态准确
4. **灵活性**：支持多种负载均衡策略
5. **可扩展性**：支持水平扩展，适应不同规模的部署

该方案适用于分布式游戏服务器架构，能够有效管理服务实例，提高系统的可靠性和可用性。
